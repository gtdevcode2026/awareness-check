Feed keyword scoring

Same methodology as the production TPRM feed stack (weighted scoring + CVE id exclusion). Each RSS item uses title + summary text. If it matches CVE-YYYY-NNNN, the item is dropped. Otherwise: each critical phrase +5, each context phrase +2, each noise phrase −3. Items with score ≥ 5 are kept. Built-in defaults emphasize employee awareness on the critical list (phishing, scams, smishing, vishing, BEC, passwords/MFA, training) and technical / vuln research on the noise list (CVE, CWE, CVSS, exploits, patches, PoCs, and similar).
